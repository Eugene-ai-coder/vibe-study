# 2단계: 요구사항 확정 (02_requirements)

## 1. 기능 명세

### 1.1 메뉴관리 화면 (MenuPage)
- **트리뷰 UI**: 좌측에 메뉴 트리(3레벨 이상 지원), 우측에 선택 메뉴의 상세 편집 폼
- **CRUD**: 메뉴 추가(하위 메뉴 포함), 수정, 삭제
- **정렬 순서**: `sort_order`로 동일 레벨 내 표시 순서 제어
- **사용 여부**: `use_yn` 플래그로 메뉴 활성/비활성 제어
- **접근 제한**: ADMIN 역할만 메뉴관리 화면 접근 가능

### 1.2 메뉴 항목 필드
| 필드 | 설명 | 필수 |
|------|------|------|
| menu_id | 메뉴 ID (PK) | Y (자동생성) |
| menu_nm | 메뉴명 | Y |
| menu_url | 메뉴 URL (라우트 경로) | N (그룹메뉴는 URL 없음) |
| parent_menu_id | 상위 메뉴 ID (NULL이면 최상위) | N |
| sort_order | 정렬 순서 | Y (기본값 0) |
| use_yn | 사용 여부 | Y (기본값 'Y') |
| menu_level | 메뉴 깊이 (1, 2, 3...) | Y (자동 계산) |

### 1.3 좌측 사이드바 메뉴 연동
- 기존 Sidebar.vue의 정적 MENU 배열을 **DB 기반 API 조회**로 전환
- 로그인 사용자의 역할에 따라 허용된 메뉴만 트리 구조로 반환
- `use_yn = 'Y'`인 메뉴만 노출
- `sort_order` 기준 정렬

### 1.4 역할(Role) 기반 메뉴 권한
- **`tb_user_role` 테이블 신규 생성**: 사용자-역할 다대다 매핑
- **`tb_menu_role` 테이블 신규 생성**: 메뉴-역할 다대다 매핑
- 역할 종류: 공통코드로 관리 (예: ADMIN, USER)
- 메뉴관리 화면에서 메뉴별 허용 역할을 지정
- 사이드바 조회 시 사용자의 역할과 매칭되는 메뉴만 반환

### 1.5 메뉴관리 메뉴 등록
- '시스템 설정' 그룹 하위에 '메뉴관리' 항목 추가
- URL: `/menu`

### 1.6 초기 데이터
- 현행 사이드바 메뉴를 `tb_menu`에 INSERT
- 기존 사용자에게 기본 역할(ADMIN) 부여

## 2. 비기능 요건
- 메뉴 트리 조회 API 응답 시간: 200ms 이내
- 사이드바 메뉴는 로그인 시 1회 조회 후 Pinia 스토어에 캐싱
- API 호출 실패 시 에러 메시지 표시 (빈 메뉴 방지)

## 3. 수용 기준
- [ ] 메뉴관리 화면에서 메뉴 CRUD가 정상 동작한다
- [ ] 트리뷰에서 3레벨 이상의 메뉴 계층이 표시된다
- [ ] 메뉴 추가/수정/삭제 후 사이드바에 즉시 반영된다
- [ ] 하위 메뉴가 있는 메뉴 삭제 시 차단된다
- [ ] 역할별 메뉴 권한이 사이드바에 정상 적용된다
- [ ] ADMIN이 아닌 사용자는 메뉴관리 화면에 접근 불가하다
- [ ] 기존 메뉴가 초기 데이터로 빠짐없이 등록된다

## 4. 제외 범위
- 메뉴 아이콘 기능 (텍스트만 사용)
- 드래그 앤 드롭 정렬 (sort_order 수동 입력)
- 역할 관리 CRUD 화면 (공통코드로 관리)
- 메뉴 URL 유효성 자동 검증

## 5. 용어 정의
| 용어 | 정의 |
|------|------|
| 메뉴(Menu) | 사이드바에 표시되는 네비게이션 항목 |
| 그룹 메뉴 | URL 없이 하위 메뉴를 묶는 상위 메뉴 |
| 리프 메뉴 | 실제 페이지로 이동하는 최하위 메뉴 |
| 역할(Role) | 사용자에게 부여되는 권한 단위 (ADMIN, USER 등) |
