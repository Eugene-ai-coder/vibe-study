## 1. 기능 명세

### 1.1 전체 레이아웃 통일
- 로그인 화면을 제외한 모든 메인화면에 `MainLayout`(헤더 고정 + LNB + 본문) 적용
- 대상: `SubscriptionPage`, `BillStdPage`, `StudyLogPage` → `Layout` → `MainLayout` 교체

### 1.2 전체 UI 밀도 조정
- 목록 행 높이: `h-7`(28px), 텍스트: `text-sm`
- 입력 컴포넌트(input, select, button) 높이: `h-8`(32px)
- 적용 범위: 전체 화면 공통 (신규 화면 포함)

### 1.3 메뉴 추가
- 위치: 가입관리 그룹 > 과금기준 하위에 "대표가입 관리" 추가
- 연결 경로: `/subscription-main`

### 1.4 가입별 대표가입 관리 화면 (신규)

#### 화면 구조 (4단)
| 영역 | 구성 |
|---|---|
| A. 조회조건 | 서비스 콤보 + 검색유형 콤보 + 조회조건 필드 + 조회 버튼 |
| B. 목록 | 가입별 대표가입 목록 (5개 컬럼) |
| C. 조회/입력 | 가입별대표가입 상세 정보 입력 영역 |
| D. 액션바 | 저장 버튼 (플로팅, fixed bottom-0) |

#### [영역 A] 조회조건 상세
- **서비스 콤보** 선택값 및 DB 매핑:

| 표시값 | DB 조건 (svc_nm) |
|---|---|
| 전체 | 조건 없음 |
| IDC 전력 | `= '서비스1'` |
| IDC NW | `= '서비스2'` |
| 비즈넷 | `= '서비스3'` |

- **검색유형 콤보** 선택값: `서비스`, `상품`, `가입ID`, `대표가입ID`
- **조회조건 필드**: 검색유형에 따른 검색 키워드 입력 (LIKE 검색)
- **조회 트리거**: 조회 버튼 클릭 OR 조회조건 필드 포커스 상태에서 Enter 키

#### [영역 A] 조회 유효성 검사
| 조건 | 동작 |
|---|---|
| 서비스=전체 + 검색조건 없음 | 전체 조회 허용 |
| 서비스=전체 + 검색조건 있음 | 검색조건 기준 조회 |
| 서비스≠전체 + 검색조건 없음 | 안내 메시지 표시, 조회 불가 |
| 서비스≠전체 + 검색조건 있음 | 서비스 + 검색조건 복합 조회 |

#### [영역 B] 목록 컬럼
`가입ID` / `서비스` / `상품` / `대표가입여부` / `대표가입ID`

- 조회 SQL: `tb_subscription` LEFT OUTER JOIN `tb_subscription_main`
- JOIN 조건: 가입별대표가입이 없거나, 있는 경우 SYSDATE가 유효시작일시~유효종료일시 사이
- 행 클릭 시: 해당 정보를 [영역 C]에 표시

#### [영역 C] 입력항목 상세
| 항목 | 타입 | 편집 가능 | 조건 |
|---|---|---|---|
| 가입ID | 텍스트 | 불가 (읽기전용) | - |
| 유효시작일시 | TIMESTAMP | 불가 (읽기전용) | - |
| 유효종료일시 | TIMESTAMP | 불가 (읽기전용) | - |
| 대표가입여부 | 콤보 (Y/N, default Y) | 가능 | - |
| 대표가입ID | 텍스트 + 검색 버튼 | 조건부 | 대표가입여부='Y' → 비활성화 및 NULL, 'N' → 필수 |

- **대표가입검색 버튼**: 클릭 시 가입검색 팝업 오픈
  - 팝업: 가입ID LIKE 검색 → 목록 표시 → 행 클릭 시 대표가입ID 셋팅 후 팝업 닫힘
  - 팝업 컴포넌트: `components/common/SubscriptionSearchPopup.jsx` (재사용 가능)

### 1.5 저장 로직 (대표가입정보 저장)

```
1. 현재 유효 레코드 조회 (가입ID 일치 + SYSDATE between 유효시작일시 and 유효종료일시)
2. 조회 결과 없음 → INSERT (신규)
3. 조회 결과 있음 →
   3-1. 기존 레코드: 유효종료일시 = SYSDATE 로 UPDATE
   3-2. 신규 레코드: [영역 C] 값으로 INSERT
※ 전체 @Transactional 처리
```

**저장 유효성 검사**:
- 대표가입여부='N' 이면서 대표가입ID가 비어있으면 저장 불가
- 대표가입ID는 `tb_subscription`에 실존하는 가입ID여야 함

## 2. 비기능 요건

| 항목 | 요건 |
|---|---|
| 트랜잭션 | 저장 로직 전체 `@Transactional` 필수 |

| createdBy | 프론트엔드 로그인 사용자 ID를 request body에 포함하여 전달 |

## 3. 수용 기준 (Acceptance Criteria)

- [ ] 서비스=전체 + 조건 없음으로 조회 시 전체 목록 반환
- [ ] 서비스≠전체 + 조건 없음으로 조회 시 안내 메시지 표시
- [ ] 대표가입여부='Y' 선택 시 대표가입ID 필드 자동 비활성화 및 초기화
- [ ] 대표가입여부='N' + 대표가입ID 비어있음으로 저장 시 에러 처리
- [ ] 대표가입ID에 존재하지 않는 가입ID 입력 후 저장 시 에러 처리
- [ ] 저장 성공 시 기존 유효 레코드 종료 + 신규 레코드 INSERT 원자적 처리
- [ ] 가입검색 팝업에서 선택 시 대표가입ID 자동 셋팅 및 팝업 닫힘
- [ ] 모든 메인화면 MainLayout 적용 확인
- [ ] 목록 행 h-7(28px) 밀도 적용 확인

## 4. 제외 범위

- 가입별대표가입 삭제 기능 (이력 관리 특성상 종료 처리로 대체)
- 페이지네이션 (현행 패턴 없음, 전체 조회)
- 가입 화면 자체의 변경 (가입 삭제 시 대표가입 존재 여부 검증 추가는 포함)

## 5. 용어 정의

| 용어 | 정의 |
|---|---|
| 대표가입 | 여러 가입 중 과금 기준이 되는 대표 가입. 대표가입여부='N'인 가입은 이 가입을 참조 |
| 유효 레코드 | SYSDATE가 유효시작일시와 유효종료일시 사이에 있는 레코드 |
| 서비스 매핑 | 콤보 표시값(IDC 전력/IDC NW/비즈넷)과 DB값(서비스1/서비스2/서비스3)의 프론트엔드 코드 매핑 |
| SM + 타임스탬프 | 가입별대표가입 PK 자동 채번 패턴 (예: SM20260301143022000) |
