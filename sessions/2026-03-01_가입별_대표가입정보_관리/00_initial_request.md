## 3. 화면 및 UI 구성 (Screens & UI)
*※ 개발해야 할 복수의 화면이나 팝업을 각각의 하위 항목으로 명세합니다.*

### 3.1. 메인화면 구조 변경
- **레이아웃**
  1. 상단 헤더 + 좌측 네비게이션(LNB) + 중앙 본문 영역의 3분할 구조
  2. 상단 헤더는 스크롤하더라도 고정되어 있어야 함
  3. 위 레이아웃이 개별화면의 로직이 아니라 기본구조로 적용되도록 함. 해당 사항을 화면구성 정책에 반영 필요.
  4. 대상 : 로그인 화면을 제외한 모든 메인화면. 단, 별도로 언급하는 경우는 적용제외함.
- **줄간격 축소 및 한줄 높이 축소**: 모든 줄의 줄간격 축소, 한줄의 높이 축소. 화면에 보다 많은 정보를 담기 위함임 

### 3.2. [가입별 대표가입 관리 화면]  신규생성
- **레이아웃**: 4단 구조 : 조회조건 영역, 목록, 조회/입력항목, 액션(버튼 나열) 영역
- **영역별 구성 요소**:
  - **[영역 A] 조회조건 영역**: 
    - **서비스**: 서비스 레이블, 서비스 선택 콤보(가능값 : 전체,IDC,비즈넷),   
    - **검색유형**: 검색유형 레이블, 검색유형 선택 콤보(가능값 : 서비스,상품,가입ID,대표가입ID),조회조건 필드
    - **조회 버튼**: 검색버튼 클릭시 가입과 대표가입을 아웃터조인하여 조회정보 목록에 표시
  - **[영역 B] 목록 영역**: 
    - **가입별 대표가입 목록**: 가입ID, 서비스, 상품, 대표가입여부, 대표가입ID
  - **[영역 C] 조회/입력항목 영역**: 
    - **가입별대표가입 정보**: 
        가입ID 
        유효시작일시
        유효종료일시
        대표가입여부 : 유형은 콤보. 가능값("Y","N" default "Y")
        대표가입ID : 대표가입여부의 값이 'Y'인 경우 입력불가, 'N'인 경우 필수. 대표가입검색 버튼 추가. 가입ID를 기준으로 가입을 검색할 수 있는 가입검색화면 팝업. 가입검색화면은 가입ID을 입력하여 LIKE 검색 수행하고 조회된 목록에서 클릭하면 닫히면서 대표가입ID에 셋팅됨.
        가입검색화면은 재활용이 가능하도록 로직 적용.
  - **[영역 D] 액션 영역**: 
    - **저장 버튼**: 대표 가입정보를 가입별대표가입에 반영

## 4. 이벤트 및 비즈니스 로직 (Events & Business Logic)
### 4.1. [가입별 대표가입 관리 화면] 관련 로직
- **[가입별대표가입 조회]**: 
  - **트리거**: 조회 버튼을 누르거나 커서가 조회조건 필드에 커서가 있는 상태에서 엔터(Enter) 키를 눌렀을 때
  - **실행 로직**: 
    1. 검색유형에 따라 조회조건 필드의 값을 기준으로 가입 테이블과 가입별대표가입 테이블을 조인하여 SELECT 하여 목록에 표시.
    2. 조인조건 : 가입테이블을 기준으로 가입별대표가입 테이블을 아웃터조인함
    3. 조회조건 : 가입별대표가입 데이터가 존재하지 않거나 존재하는 경우 SYSDATE가 유효시작일시와 유효종료일시 안에 있어야 함.

  - **유효성 검사 및 예외 처리**: 
    - 조회조건 중 서비스가 전체가 아닌 경우 검색조건에 값이 있어야 조회가 가능하다.
    - 조회가 불가능할 때 공통 안내 메세지를  보여줍니다.

- **[작업대상 선택]**: 
  - **트리거**: 목록에서 특정줄을 클릭할 때
  - **실행 로직**: 
    1. 해당 정보를 조회/입력항목 영역에 표시.

  - **유효성 검사 및 예외 처리**: 
    - 특별한 추가 반영사항 없음

- **[대표가입정보 저장]**: 
  - **트리거**: 저장 버튼 클릭
  - **실행 로직**: 
    1. 가입별대표가입 정보를 조회한다.
      1.1. 가입ID 일치.
      1.2. 현재일이 유효시작일시와 유효종료일시 사이에 있음.
    2. 조회된 레코드가 없는 경우 INSERT
    3. 조회된 레코드가 있는 경우
      3.1. 조회된 레코드 UPDATE. 유효종료일시를 SYSDATE로 UPDATE.
      3.2. 조회/입력 영역의 값을 가입별대표가입 테입블에 INSERT.  

  - **유효성 검사 및 예외 처리**: 
    - 조회/입력 영역의 대표가입ID는 대표가입여부가 'Y'인 경우 입력불가, 'N'인 경우 필수입력.
    - 대표가입ID는 가입 테이블에 존재해야함.
