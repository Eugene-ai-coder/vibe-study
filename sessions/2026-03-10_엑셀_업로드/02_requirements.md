# 대표가입 관리 - 엑셀 업로드/다운로드 요구사항

## 1. 기능 명세

### 1.1. 엑셀 다운로드
- 대표가입 목록(그리드)에서 **복수 행을 체크박스로 선택** 후 다운로드
- 대상 컬럼: `가입ID`, `대표가입여부`, `대표가입ID` (3개 필드만)
- .xlsx 형식

### 1.2. 엑셀 업로드
- 다운로드와 동일한 3개 컬럼 포맷의 엑셀 파일을 업로드
- 업로드 후 그리드에 **미저장 상태**로 표시 (DB 반영 전)
- 가입ID 기준 병합 로직:
  - 기존 그리드에 동일 가입ID 존재 → 해당 행의 값을 업로드 값으로 덮어쓰기
  - 기존 그리드에 없는 가입ID → 새 행으로 추가
- 유효성 검증:
  - 엑셀의 가입ID가 SUBSCRIPTION 테이블에 존재하지 않는 경우 → 그리드에 추가하되 `처리결과` 컬럼에 에러 메시지 표시
  - 에러 행은 체크박스 선택 불가 (저장 대상에서 제외)

### 1.3. 변경 여부 표시
- 그리드에 `변경여부` 상태 컬럼 추가
- 원본(DB) 값과 현재 값을 비교하여 변경된 행에 시각적 표시

### 1.4. 선택 저장
- 그리드에서 복수 행을 체크박스로 선택 후 저장
- 선택된 행만 SUBSCRIPTION_MAIN 테이블에 반영 (벌크 저장)

### 1.5. 처리결과 컬럼
- 그리드에 `처리결과` 컬럼 추가
- 업로드 시 유효성 에러 메시지 표시
- 저장 성공/실패 결과 표시

## 2. 비기능 요건
- Apache POI 의존성 추가 필요
- Multipart 파일 업로드 설정 필요
- DataGrid 공통 컴포넌트에 체크박스 선택 기능 확장

## 3. 수용 기준
- 엑셀 다운로드: 선택한 행의 3개 필드가 .xlsx로 정상 다운로드
- 엑셀 업로드: 파일 업로드 후 그리드에 미저장 상태로 병합/추가 표시
- 유효성 검증: 존재하지 않는 가입ID는 에러 표시 + 선택 불가
- 변경 여부: 원본 대비 변경된 행에 상태 표시
- 선택 저장: 체크된 행만 DB에 벌크 저장 성공

## 4. 제외 범위
- 엑셀 양식 템플릿 별도 다운로드 기능
- 엑셀 업로드 시 대용량(1만건 이상) 처리 최적화
- 엑셀 파일 형식 검증 (컬럼 순서/명칭 불일치 등 상세 검증)

## 5. 용어 정의
- **대표가입**: SUBSCRIPTION_MAIN 테이블의 레코드
- **가입**: SUBSCRIPTION 테이블의 레코드
- **병합**: 가입ID 기준으로 기존 행 덮어쓰기
- **미저장 상태**: DB에 반영되지 않고 프론트엔드 상태로만 존재
