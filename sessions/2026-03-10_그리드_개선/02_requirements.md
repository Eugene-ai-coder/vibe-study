# 그리드 개선 요구사항

## 1. 기능 명세

### F1. 그리드 페이징 추가
- **대상 화면**: 가입 관리(`SubscriptionPage`), 대표가입 관리(`SubscriptionMainPage`), 특수가입 관리(`SpecialSubscriptionPage`)
- **기준**: UserPage와 동일한 방식의 서버사이드 페이징
- **변경 내용**: 현재 전건 조회 → 페이징 조회로 전환 (백엔드 API + 프론트엔드 DataGrid props)

### F2. 콤보 글자 깨짐 수정
- **현상**: 콤보박스(셀렉트)에서 글자가 모두 깨져 보임
- **조치**: 런타임 원인 분석 후 수정 (CommonCodeSelect 또는 useCommonCodeLabel 관련)

### F3. 가입 테이블 필드 변경 (서비스명/요금상품명 → 서비스코드/요금상품코드)
- **대상**: `tb_subscription` 테이블
- **변경**: `svc_nm`(서비스명) → `svc_cd`(서비스코드), `fee_prod_nm`(요금상품명) → `fee_prod_cd`(요금상품코드)
- **공통코드 추가**: `fee_prod_cd` 그룹 신규 생성, 항목: 요금상품 A ~ F (6개)
- **영향 범위**: Entity, DTO, Repository(네이티브 쿼리 포함), ServiceImpl, DataInitializer, 프론트엔드 3개 화면

### F4. "코드" 접미사 제거
- **대상**: 모든 화면의 그리드 헤더 및 폼 레이블에서 필드명이 "코드"로 끝나는 항목
- **변경**: "서비스코드" → "서비스", "요금상품코드" → "요금상품" 등
- **제외**: 공통코드 관리 화면 (코드 자체를 다루는 화면이므로)

### F5. 대표가입 관리 subsNm 버그 수정
- **현상**: `SubscriptionMainPage`의 `subsNm`(가입명) 컬럼이 DTO/쿼리에 없어 항상 빈값
- **조치**: DTO와 네이티브 쿼리에 `subs_nm` 필드 추가하여 정상 표시

## 2. 비기능 요건
- H2 인메모리 DB 기반이므로 `schema.sql`, `data.sql` 직접 수정
- 네이티브 쿼리는 Entity 변경과 반드시 동기화
- 프론트엔드 빌드 산출물 재생성 불필요 (개발 환경)

## 3. 수용 기준
- [ ] 세 화면(가입/대표가입/특수가입) 그리드에 페이징이 동작한다
- [ ] 콤보박스에서 한글이 정상 표시된다
- [ ] 가입 데이터의 서비스/요금상품이 코드 기반으로 저장·조회된다
- [ ] fee_prod_cd 공통코드에 6개 항목이 존재한다
- [ ] "코드"로 끝나는 필드명이 화면에서 "코드" 없이 표시된다 (공통코드 관리 제외)
- [ ] 대표가입 관리에서 가입명이 정상 표시된다

## 4. 제외 범위
- 공통코드 관리 화면의 "코드" 필드명은 변경하지 않음
- 기존 페이징 없는 동작에 대한 하위 호환성 불필요

## 5. 용어 정의
- **서비스코드(svc_cd)**: 기존 공통코드 그룹, 서비스 유형 분류
- **요금상품코드(fee_prod_cd)**: 신규 공통코드 그룹, 요금상품 A~F

## Summary (다음 단계 전달용)
- **핵심 기능**: (1) 3개 화면 페이징 추가 (2) 콤보 깨짐 수정 (3) svc_nm/fee_prod_nm → svc_cd/fee_prod_cd 변경 + fee_prod_cd 공통코드 신규 (4) "코드" 접미사 제거 (5) subsNm 버그 수정
- **수용 기준**: 페이징 동작, 콤보 한글 정상, 코드 기반 저장/조회, fee_prod_cd 6개 항목 존재, "코드" 제거 확인, 가입명 정상 표시
- **제외 범위**: 공통코드 관리 화면의 "코드" 필드명
- **비기능 요건**: schema.sql/data.sql 직접 수정, 네이티브 쿼리 동기화 필수
