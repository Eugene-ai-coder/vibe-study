# 2단계: 요구사항 확정 (02_requirements)

## 기능 A: 메뉴 정렬 순서 개선

### A-1. 위/아래 이동 버튼
- TreeNode에 마우스 호버 시 ▲▼ 버튼 표시 (기존 '+' 버튼 옆)
- ▲ 클릭: 같은 부모 내에서 바로 위 형제와 sortOrder 교환
- ▼ 클릭: 같은 부모 내에서 바로 아래 형제와 sortOrder 교환
- 첫 번째 노드의 ▲, 마지막 노드의 ▼는 비활성화(또는 숨김)

### A-2. 정렬 순서 자동 재정렬
- 메뉴 생성 시: 같은 부모의 마지막 순서 + 1로 자동 배치
- 메뉴 삭제 시: 같은 부모 내 sortOrder를 1, 2, 3... 순으로 자동 압축
- 이동(▲▼) 시: 두 메뉴의 sortOrder만 교환 (전체 재정렬 불필요)

### A-3. 편집 폼 변경
- sortOrder 입력 필드 완전 제거 (폼에서 숨김)
- 신규 메뉴 생성 시 sortOrder는 백엔드에서 자동 계산

### A-4. API 변경
- `POST /api/menus/{menuId}/move-up` — 위로 이동
- `POST /api/menus/{menuId}/move-down` — 아래로 이동
- 기존 create/update API에서 sortOrder를 클라이언트가 보내지 않아도 되도록 변경

## 기능 B: 권한관리 화면

### B-1. 범위
- 역할 목록은 공통코드(ROLE 그룹)에서 조회 (역할 CRUD는 공통코드관리에서 처리)
- 권한관리 화면에서는 **역할-사용자 매핑**만 관리
- 메뉴-역할 매핑은 기존 메뉴관리 화면에서 계속 처리

### B-2. UI 레이아웃 — 좌우 분할
- **좌측 패널 (30~40%)**: 역할 목록
  - 공통코드 ROLE 그룹의 상세코드 목록 표시
  - 역할 클릭 시 우측에 해당 역할의 사용자 목록 로드
  - 선택된 역할 하이라이트
- **우측 패널 (60~70%)**: 사용자 할당
  - 해당 역할에 할당된 사용자 그리드 (DataGrid 컴포넌트 사용)
  - 그리드 컬럼: 사용자ID, 닉네임, 계정상태
  - 사용자 추가 버튼 → 미할당 사용자 선택 팝업 또는 검색
  - 사용자 제거 버튼 → ConfirmDialog 후 해제
- **하단**: FloatingActionBar (저장/취소)

### B-3. 백엔드 API
- `GET /api/roles` — 역할 목록 조회 (공통코드 ROLE 그룹)
- `GET /api/roles/{roleCd}/users` — 특정 역할의 사용자 목록
- `PUT /api/roles/{roleCd}/users` — 특정 역할의 사용자 목록 저장 (전체 교체)
- `GET /api/roles/available-users?roleCd={roleCd}` — 해당 역할 미할당 사용자 목록

### B-4. 보안
- 권한관리 API는 ADMIN 권한 필요
- SecurityConfig에 `/api/roles/**` 엔드포인트 추가

### B-5. 메뉴/라우팅 등록
- MenuDataInitializer에 '권한관리' 메뉴 추가 (시스템설정 하위)
- Vue Router에 RolePage 라우트 추가
- 사이드바에 자동 반영 (기존 메뉴 트리 로직 활용)

## 비기능 요건
- 기존 프로젝트 규칙 준수: No Lombok, Interface+Impl, EMPTY_FORM 패턴, DataGrid 등
- 메뉴 이동 시 트리 새로고침 후 기존 선택/펼침 상태 유지

## 수용 기준
- [ ] 메뉴 트리에서 ▲▼ 버튼으로 같은 레벨 내 순서 이동 가능
- [ ] 메뉴 생성/삭제 시 sortOrder 자동 관리
- [ ] 편집 폼에 sortOrder 입력 필드 없음
- [ ] 권한관리 화면에서 역할별 사용자 조회/추가/제거 가능
- [ ] ADMIN만 권한관리 접근 가능

## 제외 범위
- 역할 CRUD (신규 역할 생성/삭제) — 공통코드관리에서 처리
- 메뉴별 세분화 권한 (조회/수정/삭제 개별 권한)
- 드래그 앤 드롭 정렬
