# 가입 관리 및 이력 조회 모듈 신규 개발

## 1. 개요
- **목적**: 로그인 기능 구현
- **개발 범위**: 신규 테이블 1개(`사용자`) 및 신규 화면 2개(로그인 화면, 종량가입관리 메인 화면) 개발

## 2. 데이터 구조 (Data Models)

### 2.1. `사용자` 테이블
- **유형/관계**: 신규 테이블. 다른 테이블과 관계 미존재.
- **항목 명세**:
| 항목명 | 타입/Key | 필수 | 설명 및 기본값 |
|---|---|---|---|
| 아이디 | VARCHAR(50) (PK) | Y | 사용자가 직접 입력하는 로그인 ID (user_id) |
| 비밀번호 | VARCHAR(255) | Y | 암호화된 비밀번호 (Hash 값 저장) |
| 이메일 | VARCHAR(100) (Unique) | Y | 이메일 주소 (비밀번호 찾기 등 활용) |
| 계정상태 | TINYINT | Y | 계정 상태 (1: 활성, 0: 탈퇴, 2: 정지 / 기본값: 1) |

## 3. 화면 및 UI 구성 (Screens & UI)

### 3.1. [로그인 화면] (사용자 인증 화면)
- **레이아웃**: 화면 중앙 집중형 박스 레이아웃 (Center-Aligned Box)
- **영역별 구성 요소**:
  - **[입력 영역]**: 
    - **아이디(user_id)**: 텍스트 입력 필드 (필수 입력, 50자 제한)
    - **비밀번호(password)**: 마스킹 처리된 텍스트 입력 필드 (필수 입력)
    - **상태 유지**: '아이디 저장' 체크박스
  - **[액션 영역]**: 
    - **로그인 버튼**: 클릭 시 `user_id`와 `password` 유효성 검증 및 세션 생성
    - **계정 찾기**: 아이디/비밀번호 분실 시 이동 가능한 텍스트 링크
  - **[하단 영역]**: 
    - **회원가입**: 계정이 없는 사용자를 위한 가입 페이지 이동 버튼

### 3.2. [종량가입관리 메인 화면] (관리 대시보드)
- **레이아웃**: 상단 헤더 + 좌측 네비게이션(LNB) + 중앙 본문 영역의 3분할 구조
- **영역별 구성 요소**:
  - **[영역 A] 헤더 (Header)**: 
    - **시스템 로고**: 클릭 시 메인 화면 리프레시
    - **사용자 정보**: 로그인한 사용자의 `닉네임` 및 `계정상태` 표시
    - **유틸리티**: 로그아웃 버튼, 알림 알람 아이콘
  - **[영역 B] 네비게이션 (Navigation)**: 
    - **가입관리 메뉴**: 가입관리, 과금기준, 특수가입관리
    - **시스템 설정**: 사용자관리,공통코드관리
  - **[영역 C] 대시보드 (Main Content)**: 
    - **현황 지표**: 전체 가입자 수, 상태별 요약 카드
    - **주요 공지**

## 4. 이벤트 및 비즈니스 로직 (Events & Business Logic)

## 4. 이벤트 및 비즈니스 로직 (Events & Business Logic)

### 4.1. [로그인 화면] 관련 로직
- **[로그인 시도]**: 
  - **트리거**: 로그인 버튼을 누르거나 키보드의 엔터(Enter) 키를 눌렀을 때
  - **실행 로직**: 
    1. 입력한 **아이디**가 등록된 정보인지 확인합니다.
    2. 해당 아이디의 **비밀번호**와 입력한 내용이 일치하는지 대조합니다.
    3. 일치하면 사용자의 **닉네임**과 **계정상태** 정보를 불러와 메인 화면으로 이동시킵니다.
  - **유효성 검사 및 예외 처리**: 
    - 아이디나 비밀번호를 입력하지 않고 로그인을 시도하면 경고창을 띄웁니다.
    - **계정상태**가 '탈퇴'나 '정지'인 경우, 로그인을 차단하고 안내 문구를 보여줍니다.
    - 정보가 틀렸을 때 보안을 위해 "아이디 또는 비밀번호가 일치하지 않습니다"라는 공통 메시지를 보여줍니다.

- **[아이디 기억하기]**: 
  - **트리거**: '아이디 저장' 체크박스를 선택하고 로그인을 성공했을 때
  - **실행 로직**: 다음에 로그인 화면에 접속했을 때 입력창에 현재 **아이디**가 자동으로 채워져 있도록 설정합니다.
  - **유효성 검사 및 예외 처리**: 체크를 해제하고 로그인하면 저장되어 있던 아이디 정보를 삭제합니다.

### 4.2. [종량가입관리 메인 화면] 관련 로직
- **[현황판 데이터 자동 업데이트]**: 
  - **트리거**: 메인 화면에 처음 들어오거나 새로고침을 할 때
  - **실행 로직**: 
    1. 현재 가입된 사용자들의 **계정상태**별 인원수를 계산하여 화면에 표시합니다.
    2. 이번 달에 발생한 서비스 사용량 데이터를 모아서 그래프로 그려줍니다.
    3. 요금을 아직 내지 않은 '미납' 상태의 가입자 명단을 찾아와서 상단에 보여줍니다.
  - **유효성 검사 및 예외 처리**: 정보를 가져오는 데 실패하면 숫자 '0'이나 '확인 불가' 문구를 표시합니다.

- **[미납자 관리 페이지 이동]**: 
  - **트리거**: 화면에 떠 있는 미납 알림 리스트에서 특정 가입자를 클릭했을 때
  - **실행 로직**: 해당 가입자의 상세 정보를 확인하고 관리할 수 있는 화면으로 연결해 줍니다.

- **[접속 종료 및 나가기]**: 
  - **트리거**: 로그아웃 버튼을 누르거나, 아무 작업 없이 오랜 시간이 지났을 때
  - **실행 로직**: 현재 로그인된 기록을 지우고 다시 [로그인 화면]으로 화면을 돌려보냅니다.
  - **유효성 검사 및 예외 처리**: 로그아웃 시 오류가 나더라도 보안을 위해 무조건 초기 화면으로 이동시킵니다.
